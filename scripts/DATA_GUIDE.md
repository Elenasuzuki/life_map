# 実データ差替ガイド

## 1. 洪水浸水想定区域（A31）

### ダウンロード

1. [国土数値情報ダウンロードサービス](https://nlftp.mlit.go.jp/ksj/) を開く
2. 左カラム「国土（水・土地）」→「**A31 洪水浸水想定区域**」をクリック
3. 都道府県「**23 愛知県**」を選択
4. 最新年度の GML ファイルをダウンロード（例: `A31-22_23_GML.zip`）
5. 解凍すると以下のような構成になる:
   ```
   A31-22_23_GML/
   ├── A31-22_23.xml       ← これが主ファイル
   ├── A31-22_23.xsl
   └── ...
   ```

### 変換（Python スクリプト）

```bash
# geopandas をインストール（初回のみ）
pip install geopandas

# convert_flood.py の INPUT_GML を書き換える
# 例:
# INPUT_GML = "/Users/elenasuzuki/Downloads/A31-22_23_GML/A31-22_23.xml"

python3 scripts/convert_flood.py
```

スクリプトが自動的に:
- 座標系を WGS84 (EPSG:4326) に変換
- 対象エリア（安城〜豊橋）でクリップ
- 浸水ランク → `depth_m` (m) に変換
- `public/data/flood_hazard.geojson` に書き出し

### 属性のマッピング

| 国土数値情報の属性 | 意味 | 変換後の属性 |
|---|---|---|
| A31_003 = "1" | 浸水深 0〜0.5m | depth_m = 0.25 |
| A31_003 = "2" | 浸水深 0.5〜3m | depth_m = 1.5 |
| A31_003 = "3" | 浸水深 3〜5m | depth_m = 4.0 |
| A31_003 = "4" | 浸水深 5〜10m | depth_m = 7.5 |
| A31_003 = "5" | 浸水深 10m以上 | depth_m = 12.0 |
| A31_007 = "1" | 計画規模 | category = "flood_l1" |
| A31_007 = "2" | 想定最大規模 | category = "flood_l2" |

### 表示色（変換後の depth_m に対応）

| depth_m | 色 | 意味 |
|---|---|---|
| < 1.0 | 黄橙 | 0.5〜1m |
| 1.0〜3.0 | オレンジ | 1〜3m |
| 3.0〜5.0 | 赤 | 3〜5m |
| 5.0 以上 | 濃赤 | 5m 以上 |

## 3. 地価コロプレス（L01 + N03）

### 概要

市区町村別の住宅地 ¥/m² 平均値をポリゴン塗り分けで表示するレイヤー。

### ダウンロード

1. [国土数値情報ダウンロードサービス](https://nlftp.mlit.go.jp/ksj/) を開く

2. **L01 地価公示** → 愛知県(23) → 最新年度の GML をダウンロード
   - 例: `L01-24_23_GML.zip` → `scripts/data/L01-24_23_GML/` に解凍

3. **N03 行政区域** → 愛知県(23) → 最新年度の GML をダウンロード
   - 例: `N03-20240101_23_GML.zip` → `scripts/data/N03-20240101_23_GML/` に解凍

### 変換・生成

```bash
pip install geopandas   # 初回のみ

python3 scripts/generate_land_price.py
```

スクリプトが自動的に:
- L01 GML を読み込み、住宅地（用途区分 = 住宅地）のみ抽出
- 市区町村コードで集計（平均・中央値・サンプル数）
- N03 の境界ポリゴンと結合
- `public/data/land_price.geojson` に書き出し

### 表示色（avgPrice プロパティを使用）

| 価格帯 ¥/m² | 色 |
|---|---|
| 〜3万 | 薄黄 |
| 3〜6万 | 黄 |
| 6〜10万 | 濃黄 |
| 10〜20万 | オレンジ |
| 20〜40万 | 濃オレンジ |
| 40万〜 | 深赤 |

### 属性マッピング（L01 → GeoJSON）

| 国土数値情報 | 意味 | 出力属性 |
|---|---|---|
| L01_006 | 価格（¥/m²） | avgPrice（集計後） |
| L01_024 | 市区町村コード | city_code |
| L01_048 | 用途区分 | フィルタに使用（住宅地のみ） |
| N03_007 | 行政区域コード | city_code（join key） |
| N03_004 | 市区町村名 | city_name |

## 4. ファイルサイズが大きすぎる場合

愛知県全体のデータはファイルサイズが大きくなることがある（洪水で 20MB超になることも）。
以下の方法で軽量化できる。

### 方法A: 範囲をさらに絞る（スクリプト内の BBOX を変更）

```python
# 安城市〜岡崎市〜豊田市付近だけ
BBOX = (137.0, 34.85, 137.25, 35.05)

# 名古屋〜豊橋の広域
BBOX = (136.7, 34.7, 137.5, 35.2)   # デフォルト
```

### 方法B: simplify の tolerance を上げる

```python
# スクリプト内
gdf.geometry = gdf.geometry.simplify(tolerance=0.0005, ...)
# 0.0001 → 0.0005 くらいに上げると面積誤差が出るが軽くなる
```

### 方法C: QGIS で手動クリップ → GeoJSON 書き出し

1. QGIS でダウンロードした GML を開く
2. メニュー「ベクタ」→「空間演算ツール」→「クリップ」
3. クリップレイヤーに対象市区町村を設定
4. 「エクスポート」→「地物を名前を付けて保存」→ GeoJSON, EPSG:4326 を選択
5. `public/data/flood_hazard.geojson` に保存

## 4. よくあるトラブル

| 症状 | 原因 | 対処 |
|---|---|---|
| ポリゴンが表示されない | ファイルが空 / BBOX 外 | BBOX を広げる |
| 色が全て同じ | `A31_003` 属性が空 | スクリプトの rank_col の検出ログを確認 |
| 学校名が表示されない | `A27_004` 属性名が違う | `gdf.columns` をスクリプトで確認 |
| 起動時にブラウザが固まる | GeoJSON が大きすぎる (>10MB) | 上記 方法A/B/C で軽量化 |
| `CRSError` が出る | 座標系情報がない | `gdf = gdf.set_crs(epsg=6668)` を追加 |
